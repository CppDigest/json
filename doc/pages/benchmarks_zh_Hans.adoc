= 基准测试 本节将Boost.JSON的性能与两个广泛使用的具有类似功能的库进行比较：以性能著称的RapidJSON和以功能丰富著称的现代C++的JSON库（nlohmann）。bench程序测量解析和序列化一组代表典型工作负载的JSON的吞吐量。评估了以下实现：

.Implementations
|===
|Name|Description

| *boost(pool)*
| Parses the input using a <<ref_monotonic_resource>>,
该资源针对无需后续修改的解析进行了优化。&lt;<ref_stream_parser>&gt;对象在试验之间重复使用，使得实现分配的临时内存得以保留，从而提高性能。</ref_stream_parser>

| *boost*
| Parses the input using the <<default_memory_resource, default memory
&lt;<ref_stream_parser>&gt;对象在试验之间重复使用，使得实现分配的临时内存得以保留，从而提高性能。</ref_stream_parser>

| *rapidjson(pool)*
| Parses the input using an instance of
https://rapidjson.org/classrapidjson_1_1_memory_pool_allocator.html[`MemoryPoolAllocator`]，该分配器针对无需后续修改的解析进行了优化。保存临时内存的https://rapidjson.org/classrapidjson_1_1_generic_document.html[`Document`]对象在试验之间不重复使用，否则内存消耗将无限增长，导致基准测试无效。

| *rapidjson*
| Parses the input using an instance of
https://rapidjson.org/classrapidjson_1_1_crt_allocator.html[`CrtAllocator`]，该分配器使用标准C++分配器，专为通用用途设计，包括解析后对文档的修改。保存临时内存的https://rapidjson.org/classrapidjson_1_1_generic_document.html[`Document`]对象在试验之间不重复使用，否则内存消耗将无限增长，导致基准测试无效。

| *nlohmann*
| Parses the input using the static member function
https://nlohmann.github.io/json/classnlohmann_1_1basic__json_ab330c13ba254ea41fbc1c52c5c610f45.html[`json::parse`]，该函数使用默认的`std`分配器，专为通用用途设计，包括解析后对文档的修改。该库未提供接口以在后续操作中重用解析或序列化期间使用的临时存储。|===

== 方法学
首先加载所有输入文件。然后，每个配置运行足够多的试验，持续至少5秒。耗时、调用次数（解析或序列化）以及传输的字节数作为样本输出，同时计算以兆字节/秒表示的吞吐量。每个配置生成多个样本（当前为五个）。除中位数的两个样本外，其余样本均被丢弃，剩余样本取平均值后作为基准测试结果报告。

输入文件位于bench/data目录中，其布局如下：

.Input Files
|===
|Name|Size|Description

| <<parse_apache_builds_json,*apache_builds.json*>>
| 125KB
| Data from the Apache Jenkins installation.

| <<parse_canada_json,*canada.json*>>
| 2.2MB
| The largest file, containing a large number of 2-element arrays holding
浮点坐标对。

| <<parse_citm_catalog_json,*citm_catalog.json*>>
| 1.69MB
| A large JSON with a variety of nesting, types, and lengths.

| <<parse_github_events_json,*github_events.json*>>
| 64KB
| An export of data from the Github Events API.

| <<parse_gsoc_2018_json,*gsoc-2018.json*>>
| 3.25MB
| Google Summer of Code 2018 data.

| <<parse_instruments_json,*instruments.json*>>
| 216KB
| An array of large objects.

| <<parse_marine_ik_json,*marine_ik.json*>>
| 2.91MB
| A three.js example model serialized to JSON.

| <<parse_mesh_json,*mesh.json*>>
| 707KB
| A JSON representing a 3D mesh. Contains many floating-point numbers.

| <<parse_mesh_pretty_json,*mesh.pretty.json*>>
| 1.54MB
| mesh.json with whitespace added.

| <<parse_numbers_json,*numbers.json*>>
| 147KB
| A array containing only floating-point numbers.

| <<parse_random_json,*random.json*>>
| 499KB
| A JSON with lots of Cyrillic characters.

| <<parse_twitter_json,*twitter.json*>>
| 617KB
| An export of data from Twitter's API.

| <<parse_twitterescaped_json,*twitterescaped.json*>>
| 550KB
| twitter.json with whitespace removed and non-ASCII characters replaced with
Unicode转义序列。

| <<parse_update_center_json,*update-center.json*>>
| 521KB
| An export of data from Twitter's API.
|===

测试使用的硬件：**Intel(R) Core(TM) i7-7700K CPU @ 4.20GHz**，Windows 10系统，32GB内存。

编译器和优化标志：gcc 8.1 (-O3)、clang 12.0 (-O3) 和 msvc 19.26 (/O2)。
