= 分配控制 由于<<ref_value_from>>会创建一个<<ref_value>>对象，用户可能希望控制其内存分配方式。因此，该函数具有一个可选的 <<ref_storage_ptr>> 参数，用于为结果指定{ref_memory_resource}。

[NOTE]
<<ref_value_to>> 没有类似的参数，因为该函数不会创建 <<ref_value>> 对象。

由于转换结果是通过类型为`value&amp;`的输出参数设置的，因此预期的<<ref_storage_ptr>>会被正确传播。但用户仍需注意，避免意外创建使用默认 {ref_memory_resource} 的临时对象。

例如，考虑<<custom_conversions>>节中针对`ip_address`的`tag_invoke`的替代实现。

```

void tag_invoke( const value_from_tag&, value& jv, ip_address const& addr ) { jv = array{ b[0], b[1], b[2], b[3] }; }

```
